# S-Agent Core v0.3.0 release checklist

Use this checklist before announcing S-Agent Core v0.3.0 publicly.

## Repository readiness

- [ ] Main branch contains the release commit.
- [ ] Tag `v0.3.0` exists locally.
- [ ] Working tree is clean before the final announcement.
- [x] Package versions are set to `0.3.0`.
- [x] `CHANGELOG.md` includes `v0.3.0`.
- [x] README and GitHub Action examples reference `s-agent/s-agent@v0.3.0`.
- [x] Benchmark leaderboard includes the v0.3.0 validation row.
- [x] Dogfood SemanticRules cover the new diff guard ownership rule.

## Feature readiness

- [x] CLI supports `--diff <file>`, `--diff -`, and `--diff-stdin`.
- [x] Core analysis accepts explicit unified diff input.
- [x] Diff mode reports only findings whose evidence appears on added diff lines.
- [x] Existing findings elsewhere in touched files are not treated as new PR findings.
- [x] GitHub Action exposes an optional `diff` input.
- [x] Documentation explains whole-project mode versus PR diff guard mode.

## Local validation

- [x] `pnpm install` completed.
- [x] `pnpm build` passes.
- [x] `pnpm test` passes.
- [x] `pnpm lint` passes.
- [x] `pnpm rules:validate` passes.
- [x] `pnpm analyze:self` passes.
- [x] `pnpm analyze:demo:clean` passes with exit code `0`.
- [x] `pnpm benchmark` passes.

## Validation commands

Run:

```sh
pnpm install
pnpm build
pnpm test
pnpm lint
pnpm rules:validate
pnpm analyze:self
pnpm analyze:demo:clean
pnpm benchmark
```

`pnpm analyze:demo:broken` is intentionally expected to exit with code `1`
because it contains a blocking `PROVEN` violation.

## Manual GitHub actions

After validation passes:

- Create and push tag `v0.3.0`.
- Create the GitHub Release for tag `v0.3.0`.
- Confirm the composite action example works from the tag.
- Keep pull request comments out of the v0.3.0 announcement; they remain a
  known limitation.
