# S-Agent Core v0.1.0 release checklist

Use this checklist before announcing S-Agent Core v0.1.0 publicly.

## Repository readiness

- [ ] Main branch contains the release commit.
- [x] Tag `v0.1.0` exists locally.
- [ ] Working tree is clean before the final announcement.
- [ ] Referenced design-note docs under `docs/architecture/` are included in
  the release commit.
- [x] LICENSE is present.
- [x] Security policy is present.
- [x] Issue templates are present.
- [x] PR template is present.

## Documentation readiness

- [x] README Quick Start commands match `package.json`.
- [x] README explains that `pnpm analyze:demo:clean` should exit `0`.
- [x] README explains that `pnpm analyze:demo:broken` exits `1`
  intentionally.
- [x] `OPEN_CORE.md` separates S-Agent Core from future AxiomGuard Pro and
  Enterprise features.
- [x] `CONTRIBUTING.md` includes `pnpm install`, `pnpm build`, `pnpm test`,
  `pnpm lint`, and `pnpm analyze:demo:clean`.
- [x] `CHANGELOG.md` includes `v0.1.0`.
- [x] Internal documentation links are checked.

## Local validation

- [ ] `pnpm install` completed.
- [ ] `pnpm build` passes.
- [ ] `pnpm test` passes.
- [ ] `pnpm lint` passes.
- [ ] `pnpm analyze:demo:clean` passes with exit code `0`.

## Validation commands

Run:

```sh
pnpm install
pnpm build
pnpm test
pnpm lint
pnpm analyze:demo:clean
```

`pnpm analyze:demo:clean` runs the clean demo and should exit with code `0`.
A non-zero exit from `pnpm analyze:demo:clean` is not intentional.

`pnpm analyze:demo:broken` is intentionally expected to exit with code `1`
because it contains a blocking `PROVEN` violation.

## Manual GitHub actions

After validation passes:

- Create the GitHub Release for tag `v0.1.0`.
- Configure the repository description and topics from
  `docs/community/repository-settings.md`.
- Configure branch protection for `main`.
