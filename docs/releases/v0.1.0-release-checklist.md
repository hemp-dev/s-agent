# S-Agent Core v0.1.0 release checklist

Use this checklist before announcing S-Agent Core v0.1.0 publicly.

- [x] Main branch pushed.
- [x] Tag `v0.1.0` exists.
- [x] Working tree clean before release validation.
- [x] README reviewed.
- [x] LICENSE present.
- [x] Open-core docs present.
- [x] Contribution docs present.
- [x] Security policy present.
- [x] Issue templates present.
- [x] PR template present.
- [x] Build passes.
- [x] Tests pass.
- [x] Lint passes.
- [x] Demo analysis documented.
- [ ] GitHub Release created manually.
- [ ] Repo topics configured manually.
- [ ] Branch protection configured manually.

## Validation commands

Run:

```sh
pnpm install
pnpm build
pnpm test
pnpm lint
pnpm analyze:demo
```

`pnpm analyze:demo` runs the clean demo and should exit with code `0`.

`pnpm analyze:demo:broken` is intentionally expected to exit with code `1`
because it contains a blocking `PROVEN` violation.

## Manual GitHub actions

After validation passes:

- Create the GitHub Release for tag `v0.1.0`.
- Configure the repository description and topics from
  `docs/community/repository-settings.md`.
- Configure branch protection for `main`.
